## Data model

| Table | Source CSV |
| --- | --- |
| `payments` | `powerbi-data/payments.csv` |
| `recovery_scenarios` | `powerbi-data/recovery_scenarios.csv` |
| `daily_auth_rate_trend` | `powerbi-data/daily_auth_rate_trend.csv` |
| `fraud_scored_transactions` | `powerbi-data/fraud_scored_transactions.csv` |
| `fraud_model_holdout_metrics` | `powerbi-data/fraud_model_holdout_metrics.csv` |
| `feature_importance` | `powerbi-data/feature_importance.csv` |

**Design decisions:**

- Payments and fraud snapshots are independent synthetic datasets with different transaction IDs and no verified shared user namespace. They are intentionally **not related** on `transaction_id` or `user_id`.
- `recovery_scenarios` and `daily_auth_rate_trend` are pre-aggregated imports and need no relationships to fact tables.
- Portfolio-level measures on `recovery_scenarios` filter to the `ALL_DECLINES` / `ALL_BANKS` / `ALL_BRANDS` summary rows (see `MEASURES.dax`).

## Refresh workflow

Rebuild source tables from the repo root:

```powershell
.\run.ps1 -Task all
```

Then in Power BI Desktop:

1. Open `Payments_Optimization_Dashboard.pbix`.
2. If the file was moved to a new machine: **Home â†’ Transform data â†’ Data source settings â†’ Change source**, and point each query at your local `powerbi-data/<table>.csv` path.
3. **Home â†’ Refresh**.
4. Verify KPI cards: **51,237** attempts, **12.92%** failure rate, **13.71%** same-intent recovery, **2,000** fraud test rows with **200** review flags.

## Troubleshooting

| Issue | Fix |
| --- | --- |
| Retry Recovery Rate shows wrong value | Measure must filter `decline_reason = "ALL_DECLINES"` â€” see `MEASURES.dax` |
| Any measure blank after refresh | Re-create it from `powerbi/MEASURES.dax` on the correct table |
| Fraud pie chart empty | Regenerate `fraud_scored_transactions.csv` (`.\run.ps1 -Task train`) |
| `risk_bucket` column missing | Re-run training; the column is added by `fintech.py` |
| Scenario chart shows duplicate bars | Filter the visual to `decline_reason = "ALL_DECLINES"` and a single `scenario_recovery_lift_pct` |
| `fraud_user_id` error after refresh | Re-run `prepare_powerbi_tables.py`; the compatibility alias is restored in the CSV |
